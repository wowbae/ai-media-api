Ниже ТЗ, которое можно прям скопировать в нейросетевой ассистент / подрядчику для реализации модуля очистки EXIF в твоём уже существующем генераторе контента.

***

## 1. Цель задачи

Добавить в существующий веб‑сервис генерации контента модуль локальной очистки метаданных (EXIF/XMP/IPTC/PNG‑tEXt) для изображений (JPEG/PNG) перед скачиванием или публикацией.
Очистка должна происходить целиком в браузере (client‑side), без отправки оригинальных файлов на сервер.

***

## 2. Технический стек (обязателен)

- Frontend: React + TypeScript, shadcn/ui, TanStack Query (RTK Query использовать для бэкенд‑части сервиса, но конкретно чистка изображений — на клиенте).
- Backend: Node.js (bun), Express, Prisma (PostgreSQL / другая SQL‑БД).
- Интеграция с уже существующим генератором контента (код на TypeScript/React).

***

## 3. Бизнес‑логика функции очистки

1. Пользователь загружает изображение(я) в генератор контента (как сейчас).
2. Перед тем как:
   - скачать изображение,
   - отправить его в соцсети,
   - использовать внутри проекта,
   должна быть возможность включить опцию «Очистить метаданные изображений».
3. При активированной опции:
   - оригинальный файл НЕ уходит на сервер;
   - в браузере:
     - из файла удаляются все метаданные EXIF/XMP/IPTC/PNG‑text‑chunks;
     - формируется новый «чистый» файл того же формата (JPEG → JPEG, PNG → PNG);
     - визуальное содержимое не меняется.
4. Пользователь получает на выходе уже очищенную версию (вместо оригинала или дополнительно).

***

## 4. Функциональные требования (frontend)

### 4.1. UI/UX

Сделать модуль «EXIF Cleaner» как часть уже существующего интерфейса:

- Чекбокс/тумблер в настройках экспорта:
  - Название: «Очистить метаданные (EXIF)».
  - Подпись: «Удалить геолокацию и скрытые данные из изображений, обработка только на вашей стороне».
- Дополнительный экран/диалог (по желанию):
  - Возможность перетащить изображения (drag & drop) и увидеть:
    - список найденных файлов;
    - для каждого файла — статус:
      - «Найдены метаданные, будут удалены»;
      - «Метаданные не обнаружены»;
      - «Ошибка обработки».
- Поддержка множественных файлов:
  - Например, 1–50 изображений за раз.

### 4.2. Логика на клиенте

Нейронка должна выдать готовый TypeScript‑код для React, который:

1. Принимает `File` или `Blob` (JPEG/PNG) и возвращает новый `File/Blob` без метаданных.
2. Работает **офлайн** (без запросов на сервер) для очистки:
   - EXIF,
   - XMP,
   - IPTC,
   - текстовых chunk’ов PNG.
3. Использует существующую архитектуру проекта:
   - Хуки React (например, `useExifCleaner`),
   - компоненты shadcn для UI (кнопки, switches, cards, progress).
4. Обрабатывает ошибки:
   - неподдерживаемый формат;
   - повреждённый файл;
   - недостаточно памяти в браузере (вывести понятное сообщение).
5. Производительность:
   - корректно обрабатывать изображения до 20–30 МБ;
   - не блокировать основной поток (по возможности использовать `createImageBitmap` / Web Worker — по усмотрению нейронки, но без усложнения).

## 6. Интеграция с текущим проектом

Попроси нейросеть:

1. Не менять существующую архитектуру проекта, а:

## 7. Нефункциональные требования

- Безопасность:
  - исходный файл не должен отправляться ни на какие API на стороне сервера или третьих сервисов.
- UX:
  - при включённой опции не добавлять лишних шагов пользователю;
  - очистка должна происходить прозрачно и быстро.
- Логирование:
  - по желанию: логировать на клиенте, что EXIF был очищен (для аналитики), без логирования содержимого файла.
