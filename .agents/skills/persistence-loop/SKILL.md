# Persistence loop (finish tasks)

## Goal

Всегда доводить задачу до конца без просьб «продолжай», даже если план большой.

## When to apply

Используй этот навык по умолчанию для любой инженерной задачи, где есть 4+ шага, несколько файлов, внешние зависимости, тесты/линтер, миграции, деплой, рефакторинг или отладка.

## Definition of Done (DoD)

Перед началом сформулируй DoD (1–4 пункта), чтобы было ясно, когда задача завершена.

## Default behavior

-   После получения задачи: **создай todo-list** (если шагов 3+ — обязательно). Первый пункт сразу помечай как in_progress.
-   Выполняй пункты **последовательно** и **не останавливайся** на промежуточных результатах.
-   После каждых 2–3 действий: **короткий апдейт прогресса** (1–2 предложения, без лишних деталей).
-   Если что-то блокирует выполнение: **предложи обход/альтернативу** и продолжай с тем, что возможно сделать сейчас.

## Stop conditions (only)

Останавливайся только когда выполнено одно из условий:

-   Все todo отмечены как completed
-   Либо достигнут DoD, сформулированный в начале
-   Либо есть непреодолимый блокер (нет доступа/секретов/внешних данных) — тогда перечисли блокеры и что нужно от пользователя

## Quality gates (when applicable)

После существенных правок обязательно:

-   Проверяй линтер/типизацию/тесты (если они есть в проекте)
-   Исправляй найденные проблемы перед завершением

## Execution checklist

-   [ ] Сформулирован DoD
-   [ ] Создан todo-list (3+ шага, если задача не тривиальная)
-   [ ] Выполнены шаги по очереди, без остановки на полпути
-   [ ] Регулярные апдейты каждые 2–3 действия
-   [ ] Пройдены quality gates (если применимо)
-   [ ] Все todo completed или DoD достигнут
